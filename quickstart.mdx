---

This guide walks you through setting up Verify with xUnit v3 and running your first snapshot test. For other frameworks, see [Test Frameworks](/test-frameworks).

<Steps>
  <Step title="Install the NuGet package">
    Add the Verify package for your test framework:

    <CodeGroup>

    ```bash xUnit v3
    dotnet add package Verify.XunitV3
    ```

    ```bash NUnit
    dotnet add package Verify.NUnit
    ```

    ```bash MSTest
    dotnet add package Verify.MSTest
    ```

    ```bash TUnit
    dotnet add package Verify.TUnit
    ```

    ```bash Fixie
    dotnet add package Verify.Fixie
    ```

    ```bash Expecto
    dotnet add package Verify.Expecto
    ```

    </CodeGroup>
  </Step>

  <Step title="Enable ImplicitUsings">
    Verify's examples use [Implicit Usings](https://docs.microsoft.com/en-us/dotnet/core/project-sdk/msbuild-props#implicitusings). Add the following to your test project file if it isn't already set:

    ```xml
    <ImplicitUsings>enable</ImplicitUsings>
    ```

    If you prefer not to use implicit usings, substitute `Verify()` with `Verifier.Verify()` in all examples.
  </Step>

  <Step title="Write a test">
    Given a class under test that returns a `Person` object:

    ```cs
    public static class ClassBeingTested
    {
        public static Person FindPerson() =>
            new()
            {
                Id = new("ebced679-45d3-4653-8791-3d969c4a986c"),
                GivenNames = "John",
                FamilyName = "Smith",
                Spouse = "Jill",
                Children = ["Sam", "Mary"],
                Address = new()
                {
                    Street = "4 Puddle Lane",
                    Country = "USA"
                }
            };
    }
    ```

    Write a test that calls `Verify()` with the result:

    ```cs
    public class Sample
    {
        [Fact]
        public Task Test()
        {
            var person = ClassBeingTested.FindPerson();
            return Verify(person);
        }
    }
    ```
  </Step>

  <Step title="Run the test — it fails on first run">
    Run the test. It will fail because no `.verified.` file exists yet:

    ```bash
    dotnet test
    ```

    Verify creates a `Sample.Test.received.txt` file next to your test and, if a diff tool is detected, opens it automatically to show you the serialized output.

    <Note>
      Add `*.received.*` to your `.gitignore` to keep received files out of source control:

      ```txt
      *.received.*
      ```
    </Note>
  </Step>

  <Step title="Accept the snapshot">
    Review the received output and accept it. This renames `Sample.Test.received.txt` to `Sample.Test.verified.txt`.

    The accepted file looks like this:

    ```txt
    {
      GivenNames: John,
      FamilyName: Smith,
      Spouse: Jill,
      Address: {
        Street: 4 Puddle Lane,
        Country: USA
      },
      Children: [
        Sam,
        Mary
      ],
      Id: Guid_1
    }
    ```

    Notice that `Id` is stored as `Guid_1` rather than the raw GUID value. Verify automatically scrubs non-deterministic values like GUIDs and dates so your snapshots are stable across runs.

    Commit `Sample.Test.verified.txt` to source control.

    <Tip>
      Several tools make accepting snapshots easier:

      - **[DiffEngineTray](https://github.com/VerifyTests/DiffEngine/blob/main/docs/tray.md)** — Windows system tray app with accept/reject buttons.
      - **[Rider plugin](https://plugins.jetbrains.com/plugin/17240-verify-support)** / **[ReSharper plugin](https://plugins.jetbrains.com/plugin/17241-verify-support)** — Accept snapshots directly from the test runner.
      - **[Verify.Terminal](https://github.com/VerifyTests/Verify.Terminal)** — dotnet CLI tool for bulk-accepting snapshots.
      - **Clipboard** — Copy the received content and paste it into the verified file manually.
    </Tip>
  </Step>

  <Step title="Re-run — the test passes">
    Run the tests again. Verify serializes the result and compares it to `Sample.Test.verified.txt`. Because they match, the test passes.

    ```bash
    dotnet test
    ```

    If the implementation changes in the future, Verify will show a diff between the new output and the stored snapshot. You can accept the change (update the snapshot) or reject it (fix the code).
  </Step>
</Steps>
