Verify can be extended using plugins shipped as NuGet packages. Each plugin targets one or more of Verify's extension points to add support for new file types, frameworks, or comparison strategies.

## Extension points

Plugins typically manipulate a subset of Verify's extension points:

- **Comparers** — compare non-text files (images, PDFs, binaries)
- **Converters** — split a target into component parts, then verify each part separately
- **Serializer settings** — register custom JSON converters or control serialization behavior
- **Ordering** — control the order of serialized members
- **Scrubbers** — remove or replace non-deterministic values before comparison

## Enabling plugins

### Static config via ModuleInitializer

Most plugins expose a static `Initialize()` method. Call it from a `ModuleInitializer` in your test project so it runs before any tests execute.

```cs
public static class ModuleInitializer
{
    [ModuleInitializer]
    public static void Initialize() =>
        VerifyHttp.Initialize();
}
```

### Automatic plugin detection

`VerifierSettings.InitializePlugins()` scans for and initializes all Verify plugins in one call:

```cs
public static class ModuleInitializer
{
    [ModuleInitializer]
    public static void Initialize() =>
        VerifierSettings.InitializePlugins();
}
```

It works by:

1. Scanning the current assembly's directory for assemblies matching `Verify.*.dll`
2. Loading each assembly and locating `VerifyTests.[AssemblyName]`
3. Checking the `Initialized` property — if `true`, skipping to the next assembly
4. Calling `Initialize()` on the type

### Combining specific and automatic initialization

When a plugin requires parameters or must initialize before others, call it explicitly first, then call `InitializePlugins()` to handle the rest:

```cs
public static class ModuleInitializer
{
    [ModuleInitializer]
    public static void Initialize()
    {
        VerifyNServiceBus.Initialize(captureLogs: true);
        VerifierSettings.InitializePlugins();
    }
}
```

## Writing your own plugin

A plugin is a class library that calls Verify's extension APIs at initialization time. The same extension points available to test code are available to plugins.

To implement the `Initialize` / `Initialized` pattern required by `InitializePlugins()`:

```cs
namespace VerifyTests;

public static class VerifyMyPlugin
{
    public static bool Initialized { get; private set; }

    public static void Initialize()
    {
        if (Initialized)
        {
            throw new("Already Initialized");
        }

        Initialized = true;

        // Register converters, comparers, scrubbers, etc. here
    }
}
```

The `Initialized` guard prevents double-initialization when a consumer calls both `MyPlugin.Initialize()` and `VerifierSettings.InitializePlugins()`.

A reference implementation is available at [`src/Verify.SamplePlugin/`](https://github.com/VerifyTests/Verify/tree/main/src/Verify.SamplePlugin) in the Verify repository.
